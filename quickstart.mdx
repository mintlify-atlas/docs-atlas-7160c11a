Get started with HTTPToRequests by converting Burp Suite HTTP messages into Python requests code.

## Installation

<Steps>
  <Step title="Install HTTPToRequests">
    Install the package using pip:
    
    ```bash
    pip install httpconverter
    ```
    
    Or install from source:
    
    ```bash
    git clone https://github.com/freshSauce/HTTPToRequests.git
    cd HTTPToRequests
    python setup.py install
    ```
  </Step>

  <Step title="Prepare Your HTTP Message">
    Export an HTTP request from Burp Suite as XML. Here's an example of what the file looks like:
    
    ```xml
    <?xml version="1.0"?>
    <items burpVersion="2024.11.2" exportTime="Mon Dec 30 00:06:23 CST 2024">
        <item>
            <request base64="false"><![CDATA[POST /api/login HTTP/1.1
    Host: example.com
    Content-Type: application/json
    Authorization: Bearer example_token
    
    {
        "username": "test_user",
        "password": "secure_password"
    }
    ]]></request>
        </item>
    </items>
    ```
  </Step>

  <Step title="Convert Using CLI">
    Run the converter from the command line:
    
    ```bash
    httpconverter --file example.xml --output example.py
    ```
    
    Output:
    ```
    File converted and saved to example.py
    ```
  </Step>

  <Step title="View Generated Code">
    The generated `example.py` file contains ready-to-use Python code:
    
    ```python
    import requests
    
    def request_0():
        url = "https://example.com/api/login"
        headers = {
            "Host": "example.com",
            "Content-Type": "application/json",
            "Authorization": "Bearer example_token"
        }
        json = {
            "username": "test_user",
            "password": "secure_password"
        }
        response = requests.post(url, headers=headers, json=json)
        return response
    ```
  </Step>
</Steps>

## Using as a Python Module

You can also use HTTPToRequests directly in your Python code:

<Steps>
  <Step title="Import the Module">
    ```python
    from httpconverter import HTTPConverter
    ```
  </Step>

  <Step title="Create Converter Instance">
    ```python
    converter = HTTPConverter("./examples/example.xml")
    ```
  </Step>

  <Step title="Extract and Generate Code">
    ```python
    # Basic conversion
    converter.extract(filename="output.py")
    
    # With options
    converter.extract(
        use_oop=True,           # Generate OOP-styled code
        force_dicts=True,       # Force form-data to dictionaries
        filename="output.py"
    )
    ```
  </Step>
</Steps>

## Conversion Options

### OOP-Styled Code

Use the `use_oop` flag to generate object-oriented code:

<CodeGroup>

```bash CLI
httpconverter --file example.xml --output example.py --use_oop
```

```python Python
converter.extract(use_oop=True, filename="example.py")
```

</CodeGroup>

Generated output:

```python
import requests

class ClassNameHere:
    def __init__(self):
        self.session = requests.Session()
    
    def request_0(self):
        url = "https://example.com/api/login"
        headers = {
            "Host": "example.com",
            "Content-Type": "application/json",
            "Authorization": "Bearer example_token"
        }
        json = {
            "username": "test_user",
            "password": "secure_password"
        }
        response = self.session.post(url, headers=headers, json=json)
        return response
```

### Force Dictionaries

Use the `force_dicts` flag to convert form-data into dictionary format:

<CodeGroup>

```bash CLI
httpconverter --file example.xml --output example.py --force_dicts
```

```python Python
converter.extract(force_dicts=True, filename="example.py")
```

</CodeGroup>

This is useful when working with `application/x-www-form-urlencoded` content types.

## Supported Content Types

HTTPToRequests automatically handles different content types:

| Content Type | Request Parameter | Example |
|--------------|------------------|---------|
| `application/json` | `json=` | `{"key": "value"}` |
| `application/x-www-form-urlencoded` | `data=` | `key1=value1&key2=value2` |
| `multipart/form-data` | `files=` | Form data with file uploads |

## Multiple Requests Example

HTTPToRequests can process XML files containing multiple HTTP requests:

```xml
<?xml version="1.0"?>
<items burpVersion="2024.11.2">
    <item>
        <request base64="false"><![CDATA[POST /api/login HTTP/1.1
Host: example.com
Content-Type: application/json

{"username": "test_user", "password": "secure_password"}
]]></request>
    </item>
    <item>
        <request base64="false"><![CDATA[GET /api/profile HTTP/1.1
Host: example.com
Authorization: Bearer abc123
]]></request>
    </item>
</items>
```

Generated output:

```python
import requests

def request_0():
    url = "https://example.com/api/login"
    headers = {
        "Host": "example.com",
        "Content-Type": "application/json"
    }
    json = {"username": "test_user", "password": "secure_password"}
    response = requests.post(url, headers=headers, json=json)
    return response

def request_1():
    url = "https://example.com/api/profile"
    headers = {
        "Host": "example.com",
        "Authorization": "Bearer abc123"
    }
    response = requests.get(url, headers=headers)
    return response
```
